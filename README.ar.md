**نيكسوس-ريك** هو محرك استطلاع وتقييم ثغرات معياري (Modular) صمم خصيصاً لـ **فرق الاختراق الأخلاقي (Red Teams)، صائدي الثغرات (Bug Bounty Hunters)، وباحثي الأمن السيبراني**. يدمج **أكثر من 539 تقنية متخصصة** — من الاستطلاع السلبي (OSINT) إلى الاستغلال النشط — في سير عمل آلي واحد يقوده محرك فحص ذكي يتكيف مع الهدف.

## لماذا نيكسوس-ريك؟

الفجوة بين **كتابة الكود** و **اختباره أمنياً** هي حيث تتسرب الثغرات. أدوات الاستطلاع التقليدية تعاني من:

- ❌ **أحادية الغرض** — أداة للسوب دومينات، وأخرى لتحليل الجافاسكريبت، وثالثة لفحص السحابة
- ❌ **جامدة** — تشغل نفس الفحوصات على كل هدف بغض النظر عن تقنياته
- ❌ **صاخبة** — لا تراعي وجود WAF، أو حدود الطلبات، أو تحديات الأمان
- ❌ **تقارير ضعيفة** — مخرجات نصية خام بدون أدلة منظمة

نيكسوس-ريك يحل هذه المشاكل بـ **خط أنابيب موحد (Unified Pipeline)** يبدأ ببصم الهدف (Fingerprinting)، ثم يبني خطة فحص ذكية بناءً على ما اكتشفه — يشغل فقط الموديولات المناسبة، يحترم حدود الأمان، وينتج تقارير منظمة (JSON + Markdown + PDF) مع حفظ كامل للأدلة.

### ما الذي يجعله مختلفاً؟

| الميزة | نيكسوس-ريك | الأدوات التقليدية |
|---|---|---|
| **فحص متكيف مع التقنيات** | يكتشف إطار العمل (Next.js, Laravel, ASP.NET) ويشغل فحوصات مخصصة | فحوصات عمياء |
| **محرك خطة ذكية** | يختار الموديولات بناءً على البصم الفوري | قائمة ثابتة |
| **539+ تقنية** | استطلاع، استغلال، سحابة، جوال، توثيق API، أمان البريد، وغيرها | 10–50 فحص |
| **تقارير منظمة** | JSON + Markdown + PDF مع أدلة كاملة | مخرجات نصية |
| **بوابات أمان** | كشف WAF، التعامل مع تحديات الأمان، Awareness بمعدل الطلبات | بدون Awareness |
| **ثنائي اللغة** | واجهة ووثائق بالعربية والإنجليزية | إنجليزية فقط |

## 🚀 الإمكانيات

يضم نيكسوس-ريك **29 موديول** موزعة على الاستطلاع، الاستغلال، البنية التحتية، والتحليل المخصص. كل موديول يمكن تشغيله بشكل مستقل ويساهم في تقرير موحد.

### الاستطلاع (Reconnaissance)

| الموديول | التصنيف | أبرز الميزات |
|---|---|---|
| **المسح الأساسي** | 🔍 | كشف WAF (13 محرك)، بصم التقنيات (30+ إطار)، تدقيق الرؤوس الأمنية، تحليل CSP، استخراج بيانات HTML |
| **تعداد السوب دومينات** | 🌐 | سجلات الشهادات (crt.sh)، تخمين DNS (بما في ذلك أنماط bkapi, student, teacher)، استخراج من JS، اكتشاف IP الخلفي |
| **DNS، SSL والمنافذ** | 🔒 | كشف Wildcard، تحليل شهادات SSL، مسح 25 منفذاً، فحص DNSSEC |
| **مخلفات DNS** | 🗑️ | رسم خرائط سجلات DNS قديمة، حل سجلات Cloudflare المهملة، اكتشاف سوب دومينات تاريخية |
| **استطلاع السحابة** | ☁️ | AWS S3/CloudFront، Azure Blob Storage، كشف Cloudflare، تقييم تسميم الذاكرة المؤقتة |
| **استطلاع Azure السحابي** | ☁️ | كشف Azure App Service، Front Door، كشف حظر IP (`x-ms-forbidden-ip`)، تعداد Blob Storage، مسح سوب دومينات App Service |
| **تحليل الجافاسكريبت** | 📜 | استخراج endpoints API، تحليل Webpack chunks، حصاد الأسرار/التوكنز، استخراج إعدادات AWS Amplify (AppSync, Cognito, S3) |

| **كشف Salesforce** | ☁️ | كشف المثيل عبر الرؤوس/HTML، فحص السوب دومينات، تعداد إصدارات API، فحص الـ endpoints غير الموثقة |

## 🧠 الفحص الذكي (Smart Scan)

الفحص الذكي هو جوهر نيكسوس-ريك — فهو يكيّف خط أنابيب الموديولات تلقائياً حسب كل هدف.

### آلية العمل

```
         ┌─────────────────────────────┐
         │  المرحلة 0: البصم (Fingerprint) │
         │  (المسح الأساسي - دائماً)    │
         └──────────┬──────────────────┘
                    │
                    ▼
         ┌─────────────────────────────┐
         │  التحليل: Stack، WAF،       │
         │  الرؤوس، CSP، HTML، JS    │
         └──────────┬──────────────────┘
                    │
                    ▼
         ┌─────────────────────────────┐
         │  بناء الخطة الذكية          │
         │  (29 موديول → مجموعة فرعية) │
         └──────────┬──────────────────┘
                    │
                    ▼
         ┌─────────────────────────────┐
         │  تنفيذ الخطوات المختارة     │
         │  + توسيع إذا ظهرت إشارات    │
         └──────────┬──────────────────┘
                    │
                    ▼
         ┌─────────────────────────────┐
         │  إنشاء التقارير             │
         │  (JSON + Markdown + PDF)    │
         └─────────────────────────────┘
```

### قواعد القرار

يقوم المخطط الذكي بتقييم هذه الإشارات بعد البصم:

| الإشارة | التأثير |
|---|---|
| **كشف Next.js** | تفعيل الفحص العميق لـ Next.js (مسارات البناء، الـ chunks، البروكسي) |
| **كشف Laravel** | تفعيل فحوصات Laravel (Debug، Boost، Coolify) |
| **كشف Supabase** | تفعيل اختبار RLS، تعداد RPC، تدقيق Storage |
| **كشف ASP.NET/IIS** | تفعيل استطلاع Azure السحابي، اكتشاف توثيق API |
| **كشف GraphQL/Apollo** | تفعيل Introspection وفحوصات CSRF |
| **وجود نقاط تسجيل** | تفعيل تحليل أمان التسجيل |
| **وجود endpoints API** | تفعيل فحص أمان المصادقة وتسريب PII |
| **وجود بوابات دفع في CSP** | تفعيل استطلاع بوابات الدفع |
| **مؤشرات Salesforce** | تفعيل كشف Salesforce |
| **وجود إطار JavaScript** | تفعيل تحليل JS، فحص الأسرار، فحوصات APK |
| **وجود رؤوس CORS** | تفعيل الفحص العميق لـ CORS |
| **وجود WAF / تحدي أمني** | تخطي الموديولات الثقيلة، التركيز على DNS والبنية التحتية |
| **الوضع النشط/الهجومي** | تفعيل الفحص الإداري، المنطق التجاري، الفحص الكامل للثغرات |

كل قرار يُسجل في بيانات التقرير (`smart_plan_reasons`، `smart_skip_reasons`) لتعرف بالضبط لماذا تم تضمين أو استبعاد كل موديول.

## 🚀 البدء السريع

بدء فحص تفاعلي:

```bash
python3 nexus_rec.py
```

سيطلب منك:
1. النطاق المستهدف أو URL
2. وضع الفحص: `safe` (آمن)، `active` (نشط)، أو `aggressive` (هجومي)
3. وضع التخفي (Stealth mode)
4. تصدير تقرير منقح (Redacted report)
5. تأكيد التصريح (للأوضاع النشطة/الهجومية)
6. اختيار ملف الفحص

### وضع غير تفاعلي (CI/CD)

```bash
# بصم سريع
nexus-rec example.com --modules basic --auto

# فحص ذكي كامل
nexus-rec example.com --auto

# تقييم ثغرات مصرح به
nexus-rec example.com --scan-mode active --i-have-authorization --auto

# موديولات محددة فقط
nexus-rec example.com --modules basic,js,vuln,registration --auto
```

## ⚙️ مرجع الأوامر

| الخيار | الغرض |
|---|---|
| `target` | النطاق أو URL (مثال: `example.com` أو `https://example.com`) |
| `--auto` | وضع غير تفاعلي — تخطي جميع القوائم |
| `--modules` | قائمة موديولات مفصولة بفواصل |
| `--scan-mode safe` | الوضع الافتراضي — بدون حمولات تغير الحالة |
| `--scan-mode active` | فحص نشط مصرح به |
| `--scan-mode aggressive` | فحص هجومي مصرح به (أعلى ضوضاء) |
| `--i-have-authorization` | مطلوب للأوضاع active/aggressive |
| `--stealth` | تقليل التزامن، إضافة تأخير للطلبات |
| `--redact-report` | كتابة تقرير JSON منقح للمشاركة |
| `--debug` | عرض كامل لتتبع الأخطاء |
| `--check-deps` | فحص التبعيات المطلوبة والاختيارية |

## 📊 التقارير

كل فحص ينتج مجلداً بختم زمني:

```
results/YYYYMMDD_HHMMSS-النطاق/
├── raw_report.json         # الأدلة الكاملة (قابل للتنقيح)
├── redacted_report.json    # فقط مع --redact-report
├── security_report.md      # تقرير Markdown قابل للقراءة
└── security_report.pdf     # نسخة PDF محمولة
```

التقارير تشمل:
- **ملخص تنفيذي** مع بيانات الهدف، تغطية الفحص، وقرارات الموديولات
- **التقنيات** المكتشفة مع مصادر الأدلة
- **تدقيق الرؤوس الأمنية**
- **قرارات الخطة الذكية** (لماذا تم تشغيل أو تخطي كل موديول)
- **نظرة عامة على النتائج** مع أعداد لكل موديول
- **أقسام مفصلة** لكل موديول أنتج نتائج
- **أدلة JSON خام** للأتمتة والفحص العميق

## 🔎 نموذج الأمان

| الوضع | السلوك |
|---|---|
| `safe` (افتراضي) | استطلاع للقراءة فقط؛ بدون حمولات تغير الحالة؛ يتخطى الفحص الإداري والمنطق التجاري |
| `active` | فحص ثغرات مصرح به مع حمولات؛ يتطلب `--i-have-authorization` |
| `aggressive` | فحص هجومي مصرح به (سباق التوقيت، هجمات إعادة التشغيل)؛ يتطلب `--i-have-authorization` |

- جميع النتائج الخام محفوظة محلياً
- `--redact-report` يزيل القيم الحساسة (التوكنز، المفاتيح، بيانات الدخول) للمشاركة
- كشف WAF/CDN يمكنه منع الموديولات الثقيلة تلقائياً

## 🧯 استكشاف الأخطاء

| المشكلة | السبب | الحل |
|---|---|---|
| `Vercel Security Challenge` | الهدف يعرض صفحة تحدي أمني | أعد المحاولة لاحقاً، استخدم IP مصرح به، أو شغل الموديولات السلبية |
| `Invalid module(s)` | اسم موديول غير معروف | تحقق من قائمة الموديولات الصالحة أعلاه |
| تحذير `Dependency check` | حزمة اختيارية مفقودة | شغل `pip install -r requirements.txt` |
| خطأ غير متوقع | استثناء أثناء التشغيل | أعد التشغيل مع `--debug` لعرض التتبع الكامل |

## ⚠️ إخلاء مسؤولية أخلاقي

هذه الأداة مخصصة **لاختبار الاختراق المصرح به والبحث التعليمي فقط**. يجب أن يكون لديك إذن كتابي وصريح من مالك النظام قبل استخدام نيكسوس-ريك ضد أي هدف. الاستخدام غير المصرح به غير قانوني.
